# Security Policy

## Supported Versions

We support security updates on the latest commit in `master`.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use GitHub Security Advisories for private disclosure:
1. Go to this repository's `Security` tab.
2. Click `Report a vulnerability`.
3. Include clear reproduction steps, affected versions/commits, and impact.

If GitHub Security Advisories are unavailable, open an issue titled `Security Contact Request` without sensitive details and we will provide a private channel.

## Scope

Please report vulnerabilities that could impact confidentiality, integrity, or availability, including:
- Authentication/authorization bypasses
- Token/key leakage vectors
- XSS/CSRF/injection vulnerabilities
- Privilege escalation via gateway integration

## Response Targets

- Initial triage response: within 3 business days
- Status update cadence: at least weekly until resolution
